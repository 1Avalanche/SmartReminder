# SmartReminder Chat Agent

CLI-чат-агент с поддержкой двух моделей: DeepSeek v4 Pro и Qwen3-235B-A22B-Thinking.

## Запуск

```bash
# через скрипт
./cli/run.sh

# с указанием модели
./cli/run.sh --model qwen

# напрямую через Gradle
./gradlew :cli:run --args="--model deepseek"
```

## Настройка

API-ключи задаются в `local.properties` в корне проекта:

```properties
DEEPSEEK_STUDY_API_KEY=sk-...
OPENROUTER_STUDY_API_KEY=sk-...
```

## Команды

| Команда | Описание |
|---------|----------|
| `/help` | Показать справку |
| `/models` | Список доступных моделей |
| `/model <name>` | Переключить модель (deepseek, qwen) |
| `/history`, `/hist` | Показать историю запросов |
| `/clear` | Очистить историю и лог |
| `/exit`, `/quit` | Выйти |

```
└── Colors.kt         — ANSI-константы для цветного вывода
```

## Как это работает

### System prompt
Каждый запрос содержит system-сообщение с инструкцией возвращать JSON с полями:
- `keywords` — ключевые слова из запроса
- `summary` — краткое описание запроса
- `content` — ответ для отображения пользователю

Пользователю показывается только `content`. Остальные поля сохраняются для построения контекста следующих запросов.

### Контекст
В запрос не включается полная история переписки — только system-сообщение, контекст из последних 10 ответов и текущий запрос. Контекст добавляется двумя assistant-сообщениями:
- `keywords: ...` — список уникальных ключевых слов из последних 10 ответов
- `ранее пользователь спрашивал о: ...` — summary последних 10 ответов

### Логирование
- `cli/network.log` — полный лог сетевых вызовов (URL, хэдеры, тело запроса и ответа, код статуса). API-ключ маскируется. Файл в `.gitignore`, очищается командой `/clear`.
- `/history` — история сессии в памяти: ввод пользователя, payload запроса, ответ модели.
