# Code Analyzer Mode

Режим анализа кода: поиск багов, уязвимостей, code review, рефакторинг. Технически идентичен `chat`, отличается только системным промптом.

Активируется: `/mode code-analyzer`

---

## Системный промпт

Определён в `ModelConfig.kt`:

> Ты — эксперт по анализу кода. Твои задачи: искать баги и уязвимости, предлагать рефакторинг, объяснять архитектурные решения, проводить code review.  
> При анализе указывай точные места проблем (файл:строка если известно), объясняй причину, предлагай конкретное исправление с примером кода.  
> Анализируй код структурно: сначала общий обзор, затем конкретные проблемы по приоритету.

Вся остальная инфраструктура (история, сжатие, профиль, file context) — та же, что в `chat`. Подробнее: [chat.md](chat.md).

---

## Два способа работы с кодом

### 1. Режим `/mode code-analyzer` + `/read`

Вручную загружаешь нужные файлы и задаёшь вопросы. Подходит для точечного анализа конкретных файлов.

```
/mode code-analyzer
/repo /path/to/project
/read src/auth/TokenService.kt
/read src/auth/SessionManager.kt
найди потенциальные проблемы с безопасностью
```

### 2. Команда `/analyze <path> [prompt]` (доступна в любом режиме)

Автоматически собирает все текстовые файлы из директории и отправляет на анализ одним запросом. Удобно для обзора большого модуля целиком.

```
/analyze src/auth
/analyze src/auth найди утечки памяти и проблемы с конкурентностью
```

**Как работает `/analyze`:**

1. `FileScanner` рекурсивно обходит директорию
2. Все текстовые файлы конкатенируются в один блок:
   ```
   === File: src/auth/TokenService.kt ===
   <содержимое файла>

   === File: src/auth/SessionManager.kt ===
   <содержимое файла>
   ```
3. Блок отправляется в текущую модель с промптом (или дефолтным: "Проанализируй этот код")
4. Ответ добавляется в историю сессии

**Отличие от `/read`:** `/analyze` — одноразовый запрос без сохранения файлов в file context. `/read` загружает файл и передаёт его с каждым следующим запросом.

---

## Хранение данных

Отдельного хранилища нет. Всё пишется в общую историю сессии (`context.json`), как в chat-режиме.
