# SmartReminder Chat Agent

CLI-чат-агент с поддержкой двух моделей: DeepSeek v4 Pro и Qwen3-235B-A22B-Thinking.

## Запуск

```bash
# через скрипт
./cli/run.sh

# с указанием модели
./cli/run.sh --model qwen

# напрямую через Gradle
./gradlew :cli:run --args="--model deepseek"
```

## Настройка

API-ключи задаются в `local.properties` в корне проекта:

```properties
DEEPSEEK_STUDY_API_KEY=sk-...
QWEN_STUDY_API_KEY=sk-...
```

## Команды

| Команда | Описание |
|---------|----------|
| `/help` | Показать справку |
| `/models` | Список доступных моделей |
| `/model deepseek` | Переключиться на DeepSeek |
| `/model qwen` | Переключиться на Qwen |
| `/history` | Показать историю запросов |
| `/clear` | Очистить историю |
| `/exit` | Выйти |

## Как это работает

### System prompt
Первый запрос к модели всегда содержит system-сообщение с инструкцией возвращать JSON с полями:
- `keywords` — ключевые слова из запроса
- `summary` — краткое описание запроса
- `content` — ответ для отображения пользователю

Пользователю показывается только `content`. Остальные поля сохраняются в историю для контекста следующих запросов.

### Контекст
В запрос не включается вся история переписки — только system-сообщение, контекст из последних 10 ответов и текущий запрос пользователя. Контекст добавляется двумя assistant-сообщениями:
- `keywords: ...` — список уникальных ключевых слов из последних 10 ответов
- `ранее пользователь спрашивал о: ...` — summary последних 10 ответов

Это позволяет модели помнить, о чём говорили раньше, без дублирования полной истории.

### Анимация
Во время ожидания ответа от API отображается анимация «думаю...» с бегущими точками. Вывод идёт в stderr, чтобы избежать буферизации Gradle и обеспечить отображение в реальном времени. После получения ответа строка очищается.

### Логирование
- `cli/network.log` — полный лог сетевых вызовов (URL, хэдеры, тело запроса и ответа, код статуса). API-ключ маскируется. Очищается командой `/clear`. Файл в `.gitignore`.
- `/history` — история в сессии: что ввёл пользователь, какой payload ушёл в API, что вернула модель. Хранится в формате JSON для корректного построения контекста.

## Требования

- Java 11+
- Gradle (используется wrapper)
- Android SDK (для сборки остальных модулей проекта, не обязателен для CLI)
